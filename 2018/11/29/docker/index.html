获取镜像

Docker Hub

在 Docker Hub 中：

  • OFFICIAL 代表镜像为 Docker 官方提供和维护
  • STARS 代表镜像的关注人数
  • PULLS 代表镜像被拉取的次数

可通过命令docker search tensorflow进行搜索

删除镜像

docker rmi tensorflow

创建容器

创建容器：docker create，容器别名：docker create --name tf tensorflow

启动容器：docker start tf

使用docker run --name tf tensorflow会执行创建和启动两个步骤

  • 使用run启动的容器会将控制台连接到自己身上，可以加上-d 或 —detach来让容器运行于后台。
  • 如果镜像不存在，docker会自动执行pull操作

管理容器

查看正在运行的容器：docker ps，加上 -a 或 --all 选项可以查看所有容器

CONTAINER ID、IMAGE、COMMAND、CREATED、STATUS、PORTS、NAMES分别表示容器 ID、容器所基于的镜像、容器中主程序 ( 也就是与容器生命周期所绑定进程所关联的程序 ) 的启动命令、容器的创建时间、容器状态、容器暴露出的端口、容器名称。

容器的COMMAND主要有三种：

  • Created 此时容器已创建，但还没有被启动过。
  • Up [ Time ] 这时候容器处于正在运行状态，而这里的 Time 表示容器从开始运行到查看时的时间。
  • Exited ([ Code ]) [ Time ] 容器已经结束运行，这里的 Code 表示容器结束运行时，主程序返回的程序退出码，而 Time 则表示容器结束到查看时的时间。

停止、删除容器

停止容器：docker stop tf，停止后仍然可以通过start启动

删除容器：docker rm tf，增加 -f 或 --force 选项可以强制删除

容器网络

在 Docker 网络中，有三个比较核心的概念：沙盒 ( Sandbox )、网络 ( Network )、端点 ( Endpoint )。

  • 沙盒提供了容器的虚拟网络栈，也就是之前所提到的端口套接字、IP 路由表、防火墙等的内容。其实现隔离了容器网络与宿主机网络，形成了完全独立的容器网络环境。
  • 网络可以理解为 Docker 内部的虚拟子网，网络内的参与者相互可见并能够进行通讯。Docker 的这种虚拟网络也是于宿主机网络存在隔离关系的，其目的主要是形成容器间的安全通讯环境。
  • 端点是位于容器或网络隔离墙之上的洞，其主要目的是形成一个可以控制的突破封闭的网络环境的出入口。当容器的端点与网络的端点形成配对后，就如同在这两者之间搭建了桥梁，便能够进行数据传输了。

这三者形成了 Docker 网络的核心模型，也就是容器网络模型 ( Container Network Model )。

管理网络

创建网络

docker CLI 里与网络相关的命令都以 docker network 开头，其中创建网络的命令是 docker network create。

通过-d选项可以为新的网络指定驱动的类型，其值可以是 bridge、host、overlay、maclan、none，也可以是其他网络驱动插件所定义的类型；当我们不指定网络驱动时，Docker默认采用Bridge Driver作为网络驱动 。

通过 docker network ls 或是 docker network list 可以查看 Docker 中已经存在的网络。
